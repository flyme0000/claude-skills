# 已有文档的更新规则

仅通过 `文档注释生成/SKILL.md` 引用, 不单独使用。

---

## 适用场景

当函数/类的签名发生变更且已有文档字符串时, 产出文档应反映签名增量变化。

## 参数变更的文档产出

| 参数变更 | 文档产出 |
|---------|---------|
| 新增参数 | 在已有 `:param` 块末尾追加新参数说明行 |
| 删除参数 | 移除对应 `:param` 行; 若所有 `:param` 被移除, 移除整个 `:param` 块 |
| 参数重命名 | `:param 旧名:` 替换为 `:param 新名:`, 描述内容不变 |

新增参数时在已有 `:param` 之后追加:

```python
# 源文件: 新增参数 y
def func(x: int, y: str = ",") -> str:
    pass

# 文档产出: 追加 :param y
"""
功能简述.

:param x: 数值输入
:param y: 分隔符, 默认为 ","
:return: 处理结果
"""
```

删除参数时移除对应行:

```python
# 源文件: 移除参数 flag
def func(x: int) -> str:
    pass

# 文档产出: 移除 :param flag
"""
功能简述.

:param x: 数值输入
:return: 处理结果
"""
```

参数重命名:

```python
# 源文件: attr → name
def func(name: str) -> None:
    pass

# 文档产出: :param attr → :param name
"""
功能简述.

:param name: 参数说明
"""
```

## 返回值变更的文档产出

| 返回值变更 | 文档产出 |
|-----------|---------|
| `None` → 具体类型 | 在 `:param` 之后新增 `:return:` 行 |
| 具体类型 → `None` | 移除 `:return:` 行, 若其后紧接空行则一并移除 |

## 异常变更的文档产出

| 异常变更 | 文档产出 |
|---------|---------|
| 新增异常 | 在已有 `:raises` 末尾追加, 或新建 `:raises` 块 |
| 删除异常 | 移除对应 `:raises` 行 |

## 语义变更的文档产出

函数行为、副作用或参数含义发生根本变化时, 文档应整体重写。

属于语义变更:
- 行为变化 (如从排序变为过滤)
- 副作用变化 (如从修改原对象变为返回新对象)
- 参数含义根本变化 (如 `x` 从"数量"变为"索引")
- 返回值含义变化 (如从"用户对象"变为"用户ID")

不属于语义变更 (保留文档主体, 仅更新签名相关部分):
- 参数名变更但含义不变
- 参数类型变更但含义不变 (`int` → `float`)
- 默认值变更
- 异常类型变更但触发条件不变

## 质量检查清单

- [ ] 参数新增/删除/重命名时 `:param` 已同步更新
- [ ] 返回值变更为 `None` 时 `:return` 已移除
- [ ] 返回值从 `None` 变为具体类型时 `:return` 已添加
- [ ] 语义变更时文档已整体重写
