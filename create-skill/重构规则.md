# 重构规则

Skill 重构的专用规则。仅通过 `create-skill/SKILL.md` 引用, 不单独使用。

---

## 重构流程

### 1. 全面读取

先完整读取待重构 Skill 的 SKILL.md 及其所有依赖文件。**不得在未读取的情况下直接改写。**

### 2. 评估现状

识别当前 Skill 中:

- **保留**: 功能正常、规则清晰的章节, 不做无意义改动
- **修正**: 表述模糊、边界不清、或违反设计原则的部分
- **新增**: 缺失的规则或边界条件
- **拆分**: 涉及多个独立领域的, 拆分为多个 Skill
- **移除**: 过时或错误的规则

### 3. 设计变更方案

对照 Skill 的设计原则, 明确每项变更的理由和影响范围:

| 变更类型 | 需确认的问题                |
|------|-----------------------|
| 保留   | 是否需要改善表述?             |
| 修正   | 原规则哪里不准确? 修正后的边界是什么?  |
| 新增   | 覆盖了什么缺失场景?            |
| 拆分   | 拆分后的每个 Skill 职责边界清晰吗? |
| 移除   | 移除后会不会影响其他 Skill 的引用? |

### 4. 逐项应用

按先结构后细节的顺序应用变更:

1. 先调整章节结构 (拆分、合并、重排)
2. 再修订规则内容
3. 最后更新示例和质量清单

### 5. 验证引用一致性

重构后检查:

- 当前 Skill 对其他 Skill 的引用是否仍然有效
- 其他 Skill 对当前 Skill 的引用是否需要更新
- 质量清单是否可以覆盖本次变更

---

## 版本号规则

| 变更范围            | 版本升级   | 示例            |
|-----------------|--------|---------------|
| 仅修正表述、示例或边界条件   | 修订版 +1 | 1.1.0 → 1.1.1 |
| 新增规则或章节         | 次版本 +1 | 1.0.0 → 1.1.0 |
| 规则体系重构、拆分 Skill | 主版本 +1 | 1.x.x → 2.0.0 |
| 同时涉及新增和修正       | 次版本 +1 | 合并到次版本        |

变更说明写在 `原版 → 新版` 的变更摘要中, 在输出时列出。

---

## 拆分规则

当一个 Skill 需要拆分为多个时:

- 每个拆分出的 Skill 拥有独立的目录和 frontmatter
- 原 Skill 标记为**不再使用**或在 description 中注明已被拆分
- 各新 Skill 的 version 从 1.0.0 开始
- 输出格式: 列出所有新 Skill 的文件路径和简要说明

---

## 质量检查清单

- [ ] 重构前已完整读取待修改 Skill
- [ ] 保留了功能正常的内容, 未做无意义修改
- [ ] 版本号变更与变更范围一致
- [ ] 所有交叉引用已更新
- [ ] 拆分的 Skill 之间有清晰的职责边界
- [ ] 变更摘要已记录
- [ ] 重构后的内容满足通用质量检查清单
