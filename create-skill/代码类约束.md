# 代码类约束

代码类 Skill 的特化约束。仅通过 `create-skill/SKILL.md` 引用, 不单独使用。

---

## 泛化名约束

`create-skill/SKILL.md` 的"示例的泛化约束"规定示例不得使用真实 Skill/文件/路径名。
本文件在其基础上进一步约束**代码示例内部的标识符**，禁止使用语义化的函数名、类名、参数名等：

> 判断标准不变：如果标识符能让人联想到某个具体功能或业务含义，就不够泛化。

| 场景    | 推荐名                       | 避免                               |
|-------|---------------------------|----------------------------------|
| 函数    | `func`                    | `fetch_data`, `trigger`, `parse` |
| 类     | `MyClass`                 | `Listener`, `EventHandler`       |
| 参数    | `x`, `y`, `param`, `flag` | `url`, `timeout`, `event`        |
| 属性    | `attr`, `prop`            | `event_type`, `username`         |
| 变量    | `config`, `data`, `var`   | `__event_bus__`, `user_db`       |
| 类型参数  | `T`, `E`                  | 可以保留, 如 `T` 约定为 TypeVar          |
| 标准库异常 | `ValueError`, `TypeError` | 可以保留                             |
| 魔术方法  | `__hash__`, `__eq__`      | 可以保留, 属语言内建                      |

---

## 单规则示例约束

每个示例只演示一条规则。示例中不应出现与该规则无关的内容。

```
好: 只展示 :param 的使用
不好: 同时展示 :param + :return + :raises + Example
```

### 例外: 完整格式参考

允许在一个示例中汇集所有组成部分, 但**最多一个**, 且必须标注为格式参考而非规则。

---

## 代码示例格式

每个代码示例必须:

1. 前有规则陈述 (一行说明该示例演示什么)
2. 只包含演示该规则必需的最小代码
3. 使用 `...` 压缩无关实现

```python
# 规则陈述: 返回 None 时省略 :return:

def func(x: int) -> None:
    """
    功能简述.

    :param x: 输入值
    """
```
