# write-docstring Skill

## 核心任务

为 Python 代码 (尤其是 .pyi 存根文件) 中的公共接口生成符合规范的文档字符串。

文档字符串由若干独立组成部分拼接而成。每部分有独立的规则和示例, 生成时按需组合。

## 完整格式参考

以下是一个汇集所有组成部分的完整模板, 供参考格式:

```python
def func(self, param: Type) -> ReturnType:
    """
    功能简述 (一句话说明).

    详细说明方法的行为、使用场景、注意事项等.

    :param param: 参数说明
    :return: 返回值说明
    :raises ExceptionType: 异常说明

    Example::
        func(x, y)

    Note:
        注意事项.
    """
    ...
```

实际生成时按需选取组成部分, 不需要每部分都出现。

## 组成部分规则

每条规则附带示例, 演示边界条件。

### 基础结构

**规则**: 第一行不超过 30 字, 以句号结尾。空一行后写 2-3 句详细说明。

```python
def func(self) -> None:
    """
    功能简述.

    对输入执行变换并返回结果.
    支持自定义配置项.
    """
```

### 参数与返回值

**规则**: 
- 每个参数一行 `:param x:` 说明用途和约束
- 返回值非 `None` 时必须写 `:return:`, 返回 `None` 时省略
- 异常用 `:raises ExceptionType:` 说明触发条件

```python
def func(x: int, y: str) -> str:
    """
    功能简述.

    :param x: 数值输入
    :param y: 字符串输入
    :return: 处理结果
    :raises ValueError: 参数无效时抛出
    """
```

### 附属信息

**规则**: `Example::` 后换行缩进写代码, 仅当有必要演示用法时添加。`Note:` 可选, 放在文档末尾。

```python
def func(x: int) -> int:
    """
    功能简述.

    :param x: 输入值
    :return: 计算结果

    Example::
        result = func(42)

    Note:
        该函数有副作用, 会修改全局状态.
    """
```

## 接口类型的组成差异

不同接口类型选取不同的组成部分组合。以下说明差异点, 不重复展示完整文档。

### 类文档

包含功能说明和使用场景, 可选 Example 和 Note。

```python
class MyClass(Generic[T]):
    """
    泛型容器类.

    适用于需要类型安全集合的场景.
    """
```

`Example` 和 `Note` 按需添加。

### 属性文档

属性没有参数, 不包含 `:param:`。有返回值时写 `:return:`, 返回 None 时省略。

```python
@property
def attr(self) -> str:
    """
    属性简述.

    :return: 属性值的说明
    """
```

```python
@property
def attr(self) -> None:
    """占位属性, 未实现."""
```

### 模块级变量

使用变量下方的文档块, 不包含 `:param:` 和 `:raises:`。

```python
config: dict[str, Callable[..., None]]
"""
全局配置映射表.

内部数据结构, 用于存储注册信息.
"""
```

## @overload 重载函数

**必须**同时读取本目录下的 `@overload文档规则.md`。

关键决策点概要 (完整规则见依赖文件):

| 文档组件 | 各 `@overload` 变体 | 最终实现签名 |
|----------|--------------------|-------------|
| 功能简述 | 该变体特有的简述 | — (不需要) |
| 详细说明 | — | 整体行为说明 |
| `:param`, `:return` | 该变体特有的 | — (不需要) |
| `:raises` | 该变体特有的 | 所有变体共有的 |
| `Example`, `Note` | — | 放在最终实现上 |

## 魔术方法文档判断

行为标准的魔术方法省略文档, 特殊逻辑才需要文档。

| 魔术方法 | 需要文档的条件 |
|----------|---------------|
| `__hash__` | 当不是基于 `id(self)` 或 `hash(self)` 时 |
| `__eq__` | 当不是基于 `is` 时 |
| `__repr__` | 当自定义格式时 |
| `__str__` | 当自定义格式时 |
| `__len__` | 当有特殊计算逻辑时 |
| `__iter__` | 当有特殊迭代逻辑时 |
| `__call__`, `__enter__`, `__exit__` | 作为抽象接口时, 即使行为标准也应保留文档 |

## 标点符号规范

| 规则 | 示例 |
|------|------|
| 中文与半角括号之间加空格 | `配置对象 (Config)` |
| 中文与英文之间加空格 | `类型 T 用于指定` |
| 中文与数字之间加空格 | `Python 3.12` |
| 使用半角括号 | `()` 而非 `（）` |

## 质量检查清单

- [ ] 功能简述在 30 字以内, 以句号结尾
- [ ] 返回 `None` 时省略了 `:return:`
- [ ] 返回具体类型时包含 `:return:`
- [ ] 标准行为的魔术方法省略文档
- [ ] 特殊逻辑的魔术方法包含文档
- [ ] 使用 `:param:`, `:return:`, `:raises:` (不使用 `:type:` 和 `:rtype:`)
- [ ] 文档使用中文, 使用半角标点
- [ ] 括号与中文之间加空格
- [ ] 中英文之间加空格
- [ ] 逗号后不换行
- [ ] 句号后可换行
