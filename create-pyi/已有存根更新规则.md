# 已有存根的更新规则

仅通过 `create-pyi/SKILL.md` 引用, 不单独使用。

---

## 适用场景

当源文件相对于已有 `.pyi` 或 `.py` 输出发生变更时, 产出应反映源文件的增量变化。

## 变更类型与产出结果

| 源文件变更 | `.pyi` 产出 | `.py` 产出 |
|-----------|------------|-----------|
| 新增公开函数/类 | 在对应位置新增完整存根 | 在对应位置新增去注解实现 |
| 函数签名变更 | 更新对应存根的签名行, **保留**装饰器和文档字符串 | 更新对应函数签名, **保留**实现体 |
| 删除函数/类 | 移除对应存根 | 移除对应实现 |
| 新增导入 | 新增对应导入声明 | 保持原样 |
| 删除导入 | 移除对应导入声明 | 保持原样 |

关联依据: 以函数名或类名匹配变更前后的对应关系。

## 签名变更的产出

当源文件函数签名变化时, `.pyi` 产出中对应存根的类型注解应更新, 但该存根上的装饰器和文档保持不变。

```python
# 源文件: x: int → x: float, 返回值 str → int

# .pyi 产出: 更新签名行, 保留 @final 和文档
@final
def func(x: float) -> int:
    """功能简述."""
    ...
```

## 新增存根的顺序

新增存根按源文件中的定义顺序插入。模块级函数按源文件出现顺序, 类方法按类内定义顺序。

## 文档保留条件

| 条件         | `.pyi` 文档产出                    |
|------------|--------------------------------|
| 签名变更, 语义不变 | 保留原有文档, 不做修改                   |
| 语义变更       | 按 **write-docstring** Skill 重写 |

## 质量检查清单

- [ ] 签名变更时 `.pyi` 中保留了装饰器和文档字符串
- [ ] 源文件中已删除的公开接口已从 `.pyi` 中移除
- [ ] 新增存根按源文件定义顺序插入
